Timings

NOTE: The values will be changed only inside the table. The DAG values will always be set to Engine and ms.
Also, most of the descriptions are from the official documentation so please refer to it for a more in depth explanation.

Change the timings format shown in the table.

Profiling type:
Change the profiling type.
  • store - store the values the user has selected on its knobs.
  • validate - this is where the node tells Nuke about the output it produces.
  • request - his is where the node works out what data it will need from its inputs in order to produce its output
  • engine - this is where the node does most of its work, and actually generates its output.
Call type: (Not present in DAG Inspector)
Change the call type inside the table.
  • callCount - tells you the number of times this part of the processing has been called.
  • timeTakenWall - is the time taken as it would be measured by a clock on the wall, i.e. the actual time a user would have to wait for the processing to complete.
  • timeTakenCPU - On Linux, this is the time the CPU spent executing the processing code, On Mac and Windows, the CPU time is not currently available.
Timings format:
Convert the current timings from milliseconds to the chosen value.
  • ms - milliseconds
  • s:ms - seconds milliseconds
  • m:s - minutes seconds
  • m:s:ms - minutes seconds milliseconds
